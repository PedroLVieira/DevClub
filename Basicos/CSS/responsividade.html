O que é responsividade?

Uma página responsiva é aquela que se ajusta automaticamente a diferentes tamanhos de tela (celular, tablet, notebook, monitor), sem quebrar layout, sem “estourar” conteúdo e mantendo boa leitura.

Dica: no navegador, aperte F12 e ative o modo responsivo (Device Toolbar) para testar.

Viewport (meta tag)

Para o site funcionar bem no celular, é essencial ter a meta tag do viewport. Sem ela, o navegador pode “simular” uma largura maior e o layout fica minúsculo. 

<meta name="viewport" content="width=device-width, initial-scale=1" />

Essa página já está com a tag configurada.

Unidades e tamanhos fluidos

Para responsividade, prefira medidas que se adaptam:

  • % — bom para larguras relativas (ex.: width: 100%).
  • rem — bom para tamanhos de texto e espaçamentos consistentes.
  • vw/vh — relativas ao tamanho da tela (use com cuidado).
  • clamp() — cria valores fluidos com mínimo e máximo (ótimo para tipografia).

Título com tipografia fluida (clamp)

Redimensione a janela e veja o texto ajustar sem “pular” de tamanho.

/* Tipografia fluida */
h1 {
  font-size: clamp(1.25rem, 2.5vw + 0.75rem, 2rem);
}

Media queries

Media queries aplicam estilos quando uma condição é verdadeira (normalmente, largura da tela). O mais comum é usar max-width (para telas menores) ou min-width (para telas maiores). 

/* Mobile-first (recomendado): base para celular */
.card { padding: 16px; }

/* A partir de 900px (tablet/desktop), ajusta layout */
@media (min-width: 900px) {
  .card { padding: 20px; }
}

Mobile: Portrait vs Landscape

No celular, o usuário pode usar o aparelho em portrait (em pé) ou landscape (deitado). Mesmo com a mesma largura, a altura disponível muda bastante, e isso pode afetar menus, seções “hero”, formulários e elementos com altura fixa.

Vire o celular (ou use o emulador) para ver qual cartão fica destacado.

Portrait (em pé) Normalmente: mais altura e menos largura. orientation: portrait
Landscape (deitado) Normalmente: mais largura e menos altura. orientation: landscape

Você pode aplicar estilos por orientação, mas use isso com moderação. Na prática, costuma ser melhor combinar orientação + altura para tratar o “landscape apertado” do mobile. 

/* Só pela orientação */
@media (orientation: landscape) {
  /* Ex.: reduzir espaçamentos ou esconder elementos secundários */
}

/* Mobile em landscape costuma ter pouca altura */
@media (max-height: 420px) and (orientation: landscape) {
  .hero { min-height: auto; padding: 16px; }
}
Dica importante: evite usar height: 100vh “cego” em mobile. Em alguns navegadores, barras do sistema mudam o valor de vh. Quando possível, prefira min-height ou use unidades modernas como dvh (ex.: min-height: 100dvh).

Flex e Grid (responsividade na prática)

A maior parte da responsividade vem de escolher bem o layout: Flex para alinhamentos e distribuição em um eixo; Grid para linhas/colunas.

Formulário que vira 1 coluna no celular

.form-grid { display: grid; grid-template-columns: 1fr 1fr; gap: 10px; }
@media (max-width: 560px) { .form-grid { grid-template-columns: 1fr; } }

Imagens responsivas

Regra básica: para a imagem nunca estourar o container, use: max-width: 100% e height: auto. 

img { max-width: 100%; height: auto; display: block; }
Exemplo de “imagem” fluida

Ela ocupa width: 100%, então acompanha o tamanho do card.

Quando quiser servir imagens diferentes por tamanho de tela, você pode usar <picture>: 

<picture>
  <source media="(max-width: 600px)" srcset="img/banner-mobile.jpg" />
  <img src="img/banner-desktop.jpg" alt="Banner" />
</picture>

Boas práticas

  • Mobile-first: escreva a base pensando no celular e vá melhorando para telas maiores.
  • Evite larguras fixas como width: 1200px. Prefira max-width + width: 100%.
  • Use grid/flex em vez de “empurrar” com margin/padding.
  • Teste com zoom e com teclado (acessibilidade também importa na responsividade).
/* Exemplo comum de container */
.container {
  width: 100%;
  max-width: 1040px;
  margin: 0 auto;
  padding: 0 16px;
}

Resumo

  • Viewport garante escala correta no celular.
  • Unidades fluidas e clamp() deixam o layout mais flexível.
  • Media queries ajustam o layout em pontos-chave.
  • Flex/Grid resolvem 90% dos layouts responsivos.

Próximo passo: aplicar esses padrões no seu projeto e testar em 360px, 768px e 1024px.
