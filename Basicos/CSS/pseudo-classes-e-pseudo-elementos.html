Pseudo-classes e Pseudo-elementos

Como estilizar estados (pseudo-classes) e “partes” de elementos (pseudo-elementos) no CSS.

Conceitos

Pseudo-classes estilizam um estado do elemento (ex.: quando o mouse está em cima, quando está em foco, quando está marcado).

Pseudo-elementos estilizam uma parte do elemento ou criam um “elemento virtual” (ex.: primeira letra, marcador da lista, um ::before para inserir um detalhe visual). 

/* Pseudo-classe: estado */
a:hover { text-decoration: underline; }

/* Pseudo-elemento: parte/virtual */
p::first-letter { font-size: 2em; }

Observação: hoje o padrão é usar :: para pseudo-elementos e : para pseudo-classes. Alguns pseudo-elementos ainda aceitam : por compatibilidade antiga (ex.: :before), mas prefira ::before.

Pseudo-classes (estados)

Alguns exemplos comuns:

  • :hover — quando o mouse passa por cima.
  • :active — enquanto está sendo clicado/pressionado.
  • :focus / :focus-visible — quando recebe foco (teclado, clique, etc.).
  • :checked — quando checkbox/radio está marcado.
  • :disabled — quando o campo/botão está desabilitado.
  • :first-child, :last-child, :nth-child() — “posição” entre irmãos.
  • :not() — negação (seleciona tudo menos…).
  • :focus-within — quando qualquer filho está em foco (ótimo para formulários).

Pseudo-elementos (partes / “elementos virtuais”)

Alguns dos mais usados:

  • ::before e ::after — cria conteúdo decorativo antes/depois do conteúdo real.
  • ::first-letter — estiliza a primeira letra.
  • ::first-line — estiliza a primeira linha (varia conforme a largura do texto).
  • ::selection — estiliza a seleção do usuário.
  • ::placeholder — estiliza o placeholder de inputs.
  • ::marker — estiliza o marcador de listas (<ul>/<ol>).

Boas práticas

  • Use :: nos pseudo-elementos (padrão moderno) e : nas pseudo-classes.
  • Prefira :focus-visible quando quiser mostrar outline só para navegação por teclado.
  • Evite colocar informação importante só em ::before/::after (conteúdo pode não ser lido por leitores de tela dependendo do caso).
  • Use pseudo-classes para feedback (hover/focus/disabled) e melhorar a usabilidade.

Resumo

  • Pseudo-classes = estados do elemento (ex.: :hover, :focus, :checked).
  • Pseudo-elementos = partes/virtual (ex.: ::before, ::first-letter, ::marker).

Dica: teste interações com teclado (Tab/Shift+Tab) para ver o foco funcionando.
