Link vs Button

Link (navegação): Ir para outra página

Botão (ação):

Como o <button> funciona

  • Dentro de formulário: o padrão pode ser type="submit". Defina type explicitamente.
  • Fora de formulário: use type="button" para ações de UI.
  • Desabilitar: adicione o atributo disabled para impedir cliques.
  • Acessibilidade: use texto claro; se o botão tiver apenas ícone, inclua aria-label.
<form action="/exemplo" method="post">
    <input type="text" name="nome" placeholder="Seu nome" />
    <button class="btn btn-primary" type="submit">Enviar</button>
    <button class="btn btn-secondary" type="reset">Limpar</button>
</form>

<!-- Botão com ícone e aria-label (somente ícone) -->
<button class="btn btn-icon" type="button" aria-label="Abrir menu">
    <!-- ícone via SVG/CSS -->
    Menu
</button>

Atributos úteis

  • name e value: enviados junto ao formulário ao clicar em submit.
  • autofocus: foca o botão automaticamente ao carregar a página.
  • form: associa o botão a um formulário por id mesmo estando fora dele.
  • formaction: URL para envio (sobrescreve o action do formulário).
  • formmethod: método de envio (get/post).
  • formenctype: MIME do envio (ex.: multipart/form-data para upload).
  • formnovalidate: ignora validação do formulário para esse envio.
  • formtarget: destino (_self, _blank etc.).
<form id="cadastro" action="/enviar" method="post">
    <input type="email" name="email" required />
</form>

<!-- Botão fora do formulário, mas associado por id -->
<button class="btn btn-primary" form="cadastro" type="submit"
        name="acao" value="salvar"
        formaction="/enviar-rapido" formmethod="post" formtarget="_self">
    Enviar cadastro
</button>

Button vs Input

  • <button>: pode conter HTML dentro (ícones, spans), mais flexível para UI.
  • <input type="button">: valor somente via value, menos flexível.
<button class="btn" type="button"><span>Salvar</span></button>
<input class="btn" type="button" value="Salvar" />

Exemplo: botão de ação (JavaScript)

Como prevenir envio acidental e executar uma ação de UI.

<form id="f">
    <input type="text" placeholder="Digite algo" />
    <button id="btn-acao" class="btn" type="button">Processar</button>
</form>

<script>
document.getElementById('btn-acao').addEventListener('click', function (ev) {
  // Executa sua ação sem enviar o formulário
  alert('Ação executada!');
});
</script>

Boas práticas de acessibilidade

  • Botões são ativáveis por teclado: Enter e Espaço funcionam por padrão.
  • Use rótulos claros; evite rótulos genéricos como “OK”. Prefira verbos: “Salvar”, “Enviar”.
  • Para botões de alternância, atualize aria-pressed="true|false" para refletir estado.
  • Não use role="button" em links; se for navegação, use <a> estilizado.
