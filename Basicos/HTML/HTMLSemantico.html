HTML Semântico

Semântica no HTML é escrever páginas usando tags que representam a função do conteúdo, deixando o CSS responsável pela aparência.

O que é semântica no HTML?

Por que usar?

  • Acessibilidade: leitores de tela entendem melhor a estrutura da página.
  • SEO: buscadores interpretam melhor o conteúdo.
  • Organização: o código fica mais legível e fácil de manter.

Principais tags semânticas

Estrutura da página

<header>
Cabeçalho de uma página ou de uma seção.
<nav>
Área de navegação com links principais.
<main>
Conteúdo principal (único por página).
<footer>
Rodapé de uma página ou de uma seção.

Conteúdo

<article>
Bloco independente (post, notícia, produto).
<section>
Seção temática dentro de uma página/article.
<aside>
Conteúdo complementar (ex.: links relacionados, dicas).
<figure> e <figcaption>
Imagem/ilustração com legenda.

Boas práticas

  • Use uma única tag <main> por página.
  • Prefira tags semânticas (header, nav, section, article) ao invés de usar div para tudo.
  • Use títulos em ordem (h1 → h2 → h3...) para dar hierarquia ao conteúdo.
  • Não use tags só pela aparência: a aparência deve ser definida no styles.css.
  • Sempre que possível, adicione atributos de acessibilidade (ex.: aria-label) quando fizer sentido.
