Imagens no HTML

Guia rápido para usar <img>, <figure> e imagens responsivas.

1) A tag <img>

A tag <img> exibe uma imagem na página. Ela não tem tag de fechamento.

  • src: caminho/URL da imagem.
  • alt: texto alternativo (acessibilidade e fallback se a imagem não carregar).
  • width/height: ajudam a reservar espaço e evitam “pulos” no layout (CLS). 
<img
    src="./img/logo.png"
    alt="Logo do DevClub"
    width="180"
    height="180"
/>

2) Exemplo com imagem (URL)

Exemplo usando uma imagem de placeholder (boa para testes). Troque por uma imagem do seu projeto.

3) Caminho relativo vs. URL

Use caminho relativo quando a imagem estiver no seu projeto e URL quando estiver em outro domínio. 

<!-- Caminho relativo (arquivo no projeto) -->
<img src="./img/foto.jpg" alt="Foto do projeto" />

<!-- URL (arquivo externo) -->
<img src="https://exemplo.com/imagem.jpg" alt="Imagem externa" />

4) <figure> e <figcaption>

Quando a imagem precisa de legenda, use <figure> e <figcaption>. 

<figure>
    <img src="./img/paisagem.jpg" alt="Paisagem ao pôr do sol" />
    <figcaption>Paisagem ao pôr do sol.</figcaption>
</figure>

5) Responsivo (srcset / picture)

Para servir imagens diferentes conforme o tamanho da tela, use srcset ou <picture>. 

<img
    src="./img/banner-1200.jpg"
    srcset="
        ./img/banner-600.jpg 600w,
        ./img/banner-1200.jpg 1200w
    "
    sizes="(max-width: 600px) 600px, 1200px"
    alt="Banner do site"
/>
<picture>
    <source srcset="./img/foto.webp" type="image/webp" />
    <source srcset="./img/foto.jpg" type="image/jpeg" />
    <img src="./img/foto.jpg" alt="Foto com fallback" />
</picture>

Boas práticas

  • Sempre preencha alt (ou alt="" se for decorativa).
  • Prefira formatos modernos (WebP/AVIF) quando possível.
  • Defina width e height para estabilidade do layout.
  • Otimize o tamanho do arquivo (compressão) para melhorar o carregamento.
