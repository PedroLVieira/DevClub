Vídeos no HTML

O HTML oferece a tag <video> para inserir e controlar vídeos diretamente no navegador, sem plugins.

Exemplo básico

Use controls para exibir os botões (play, pause, volume, tela cheia etc.). O ideal é fornecer mais de um formato com <source>, para melhorar compatibilidade. 

<video controls width="720">
    <source src="./assets/video-exemplo.mp4" type="video/mp4" />
    <source src="./assets/video-exemplo.webm" type="video/webm" />
    Seu navegador não suporta a reprodução de vídeo.
</video>

Atributos importantes

  • controls: mostra os controles do player (recomendado para acessibilidade e usabilidade).
  • autoplay: tenta iniciar automaticamente (muitos navegadores bloqueiam se não estiver muted).
  • muted: inicia sem som (comum com autoplay).
  • loop: reinicia ao terminar.
  • poster: imagem exibida antes do play (ou enquanto carrega).
  • preload: dica de carregamento: none, metadata ou auto.
  • playsinline: evita abrir em tela cheia automática no mobile (especialmente iOS).

Exemplo com poster, autoplay e loop

Para autoplay funcionar com maior chance, combine autoplay + muted. Use também playsinline em dispositivos móveis. 

<video
    width="720"
    autoplay
    muted
    loop
    playsinline
    poster="./assets/capa-video.jpg"
    preload="metadata"
>
    <source src="./assets/video-loop.mp4" type="video/mp4" />
</video>

Legendas (acessibilidade) com <track>

Use <track> para adicionar legendas (WebVTT). Isso melhora acessibilidade e compreensão. 

<video controls width="720">
    <source src="./assets/video-com-legenda.mp4" type="video/mp4" />
    <track
        src="./assets/legendas-pt.vtt"
        kind="subtitles"
        srclang="pt"
        label="Português"
        default
    />
</video>

Boas práticas

  • Fornecer mais de um formato (ex.: MP4 e WebM).
  • Adicionar legendas com <track>.
  • Evitar autoplay com som (tende a ser bloqueado e piora a experiência).
  • Usar tamanhos responsivos via CSS (ex.: max-width: 100%).
Dica: ajuste os caminhos em src para seus arquivos reais dentro do projeto (ex.: ./assets/).
